# 프로토타입 모형 (Prototype Model, 원형 모형)

프로토타입 모형은 사용자의 요구사항을 정확히 파악하기 위해 실제 개발될 소프트웨어의 
견본(시제)품(Prototype)을 만들어 최종 결과물을 예측하고 검증하는 개발 모형이다.

## 개요

- 시제품(Prototype)은 의뢰자와 개발자 모두에게 **공동 참조 모델**이 됨.
- 시스템의 일부 또는 전체 모형을 만들어 요구된 소프트웨어를 구현하는 과정.
- 프로토타입은 추후 구현 단계에서 사용될 **골격 코드(Skeleton Code)** 역할을 수행.
- 새로운 요구사항이 도출될 때마다 이를 반영하여 **프로토타입을 반복적으로 개선**함.
- 단기간 제작 목적이므로 비효율적인 언어나 알고리즘이 사용될 수 있음.

## 특징

- 사용자와 개발자 간 의사소통이 원활함.
- 요구사항 변경에 유연하게 대응 가능.
- 최종 소프트웨어에 대한 명확한 이해와 검증 가능.
- 초기 단계에서 구현된 코드는 최종 시스템에 그대로 사용될 수 있음.

## 장점

- 사용자 요구사항을 정확히 반영할 수 있음.
- 시스템 오류나 설계상의 문제를 조기에 발견 가능.
- 사용자의 만족도를 높일 수 있음.

## 단점

- 단기간 제작을 위해 비효율적인 코드 사용 가능.
- 문서화가 부족할 수 있음.
- 프로젝트 규모가 커지면 관리가 어려움.

## 요약

- 프로토타입 모형은 **사용자 요구사항 확인과 검증**을 목적으로 시제품을 반복 제작하는 모형이다.
- 요구사항 변경에 유연하며, 최종 소프트웨어 개발 전에 오류를 조기에 발견할 수 있음.
