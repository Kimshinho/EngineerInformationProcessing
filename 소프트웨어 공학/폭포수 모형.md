# 폭포수 모형 (Waterfall Model)

폭포수 모형은 폭포에서 한번 떨어진 물은 거슬러 올라갈 수 없듯이, 소프트웨어 개발도 이전 단계로
돌아갈 수 없다는 전제 하에 각 단계를 확실히 매듭짓고, 그 결과를 철저히 검토하여 승인 과정을
거친 후에 다음 단계로 진행하는 전통적인 개발 방법론이다.

## 개요

- 소프트웨어 공학에서 가장 오래되고 널리 사용된 전통적인 생명주기 모형으로, 고전적 생명 주기 모형이라고도 함.
- 개발 단계가 끝나야만 다음 단계로 넘어갈 수 있는 **선형 순차적 모형**.
- 모형 적용 경험과 성공 사례가 많음.
- 제품의 일부가 될 매뉴얼을 작성해야 함.
- 각 단계 종료 후에는 다음 단계를 수행하기 위한 **결과물 산출**이 필수.
- 두 개 이상의 단계가 병행하여 수행되지 않음.

## 개발 단계 (시나공 기준)

1. **타당성 검토**  
   - 기술적, 경제적, 운영적 측면에서 시스템 개발 가능성을 분석함.

2. **계획**  
   - 프로젝트 목표, 일정, 비용, 인력, 자원 등을 계획함.

3. **요구 분석**  
   - 사용자 요구사항을 수집, 분석하여 명확히 정의함.

4. **설계**  
   - 요구사항을 기반으로 시스템 구조, 데이터, 인터페이스를 설계함.

5. **구현(코딩)**  
   - 설계 문서를 바탕으로 프로그램을 작성함.

6. **시험(검사)**  
   - 단위/통합/시스템 테스트를 통해 오류를 검출하고 수정함.

7. **유지보수**  
   - 운영 중 오류 수정, 기능 개선 등 소프트웨어 유지보수 수행.

## 특징

- **단계별 순차 진행**: 한 단계가 완료되어야 다음 단계로 진행 가능.
- **문서 중심**: 각 단계 결과물이 명확히 문서화됨.
- **변경 대응 어려움**: 요구사항 변경에 유연하지 않음.
- **관리 용이**: 프로젝트 관리와 일정 예측이 쉬움.

## 장점

- 개발 과정과 산출물이 명확함.
- 이해하기 쉽고 관리가 용이함.
- 요구사항이 명확한 소규모 프로젝트에 적합함.

## 단점

- 요구사항 변경 대응이 어렵고 유연성이 낮음.
- 사용자 피드백이 개발 후반부에 반영됨.
- 오류 발견이 후반부에 집중될 수 있음.

## 요약

- 폭포수 모형은 **순차적·단계적 개발 모형**으로, 시나공 기준 개발 단계는  
  **타당성 검토 → 계획 → 요구 분석 → 설계 → 구현 → 시험 → 유지보수** 이다.
- 문서 중심이며 변경이 적고, 관리가 중요한 프로젝트에 적합함.
