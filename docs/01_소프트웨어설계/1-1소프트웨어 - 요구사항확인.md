# 1-1 소프트웨어 설계 - 요구사항 확인


## 1-1-1. 현행 시스템 분석

### 플랫폼
 애플리케이션이나 서비스를 개발하고 실행할 수 있는 기반 환경(주로 소프트웨어)

### 플랫폼 성능 분석 시 고려하는 항목
 - 경과시간(Turnaround Time)   :   작업이 완료될 때까지의 시간
 - 사용율(Utilization)         :   작업이 진행될 동안의 자원 사용률(CPU,메모리)
 - 응답시간(Response Time)     :   작업 요청에 대한 응답이 올 때까지의 시간
 - 가용성(Availability)        :   얼마나 안적적인가?(장애 가능성)
    
### 시스템
특정 기능을 수행하기 위해 다양한 구성 요소들이 상호작용하는 통합된 구조(주로 소프트웨어 + 하드웨어)

### 현행 시스템 분석 시 고려하는 항목
 - 운영체제 분석
 - DBMS 분석
   - 성능        : 얼마나 빠르고, 많은 데이터를 처리할 수 있는가?
   - 가용성      : 얼마나 안정적인 시스템인가?(장애 가능성)
   - 상호 호환성 : 다양한 운영체제와 호환 가능한가?
   - 기술 지원   : 안정적인 기술 지원이 가능한가?
   - 구축비용    : 라이선스 및 유지보수 비용
 - 네트웍크 분석  


## 1-1-2. 요구사항 확인

### 요구사항 분석
 - 소프트웨어가 무엇을 해야 하는가를 추적하여 요구사항 명세를 작성하는작업
 - 사용자의 요구를 추출하여 목표을 정하고, 어떤 방식으로 해결할 것인지 결정하는 단계
 - 소프트웨어 개발의 출발점이면서, 실직적인 첫 번째 단계(요구사항 분석 -> 설계 -> 구현 -> 테스트 -> 유지보수)

### 요구사항 분석의 특징
 - 개발 비용이 많이 드는 단계는 아님(유지바소 단계에서 비용이 가장 많이 든다)
 - 자료 흐름도, 자료 사전, 소단위 명세서 등의 문서가 효과적으로 이용될 수 있다
 - 코딩 / 개발 기술보다는 청취, 인터뷰, 중재 등의 커뮤니케이션 기술이 중요함

### 기능적 요구사항과 비기능적 요구사항
 - 기능적 요구사항은 시스템이 무엇을 해야하는지에 초점을 맞춤(기능) 
  ex) 사용자가 @ 뒤에 다른사람의 이름을 붙여 메시지를 전송하면 언급된 사람에게 별도의 알림이 가게 한다.
 - 비기능적 요구사항은 시스템이 어떻게 동작해야 하는지에 초점을 맞춤(성능,품질,보안 등) 
  ex) 시스템은 사용자가 멘션 기능을 수행했을 때 0.2초 안에 알림을 처리해야 한다.

###  데이터 흐름도(Data Flow Diagaram, DFD)
 - 데이터가 프로세스를 따라 흐르면서 어떻게 변화하는지를 보여주는 시작적 표현
 - 자료 흐름 그래프 또는 Bubble Chart로 표현하기도 함
 - 구조적 분석 기법에 이용되며, 데이터의 흐름에 가장 중심을 둠
 - 시간의 흐름을 명확히 표현할 수 없음(논리적 흐름 위주)

### 데이터 흐름도의 구성 요소
 - 처리기(Process)              : 데이터를 처리하는 활동 or 기능, O 로 표현
 - 데이터 흐름(Data Flow)       : 프로세스 간의 데이터 흐름, -> 로 표현
 - 데이터 저장소(Data Store)    : 데이터가 저장되는 장소, = 로 표현
 - 단말(Terminator)             : 프로세스의 시작과 끝, ㅁ 로 표현 

### 자료 사전(Data Dictionary)
 - 시스템이나 데이터베이스에서 사용하는 데이터 요소들의 정의, 속성, 제약 조건 및 관계를 체계적으로 정리한 문서
 - 모든 데이터 요소들을 통일된 형식으로 정의하여 요구사항 분석과 의사소통을 효율적으로 지원

### 자료 사전의 기호
 - =    : 자료의 정의,  ~는 ~로 구성되어 있다.
 - \+   : 자료의 연결, ~에는 ~랑 ~랑 ~가 있다.
 - {}   : 자료의 반복, {} 의 좌측에는 최소 반복 횟수, 우측에는 최대 반복 횟수를 표시 
          횟수 표시가 없으면 최소 0회, 최대 무한대
 - ()   : 자료의 생략 가능함을 표현
 - [|]  : 자료의 선택을 표현, ~랑 ~중 하나를 선택한다.
 - **   : 자료에 대한 설명
